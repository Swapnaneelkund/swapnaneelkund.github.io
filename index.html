Restructuring my Ecom Fullstack Project Restructuring my Ecom Fullstack Project

Hey everyone, as I mentioned earlier, I used to work at HOSLA, an NGO startup, where I made the complete back end of the mental health service and also worked on all other services. After working there and applying all my new learnings, when I looked back, I found my personal project was kind of messy — not modular, no custom errors, no custom logs, no global error handler, no async handler (to wrap the function so if an error arrives, it is sent to the global error handler) — just messy try-and-catch blocks. No controllers, just doing all the tasks in the router. I also had to add proper spacing and remove extra commas.
